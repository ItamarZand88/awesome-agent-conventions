# Security policy

This repository is a documentation and examples catalog. It does not publish a
runtime package, but security and privacy issues can still show up in copied
examples, scripts, CI, or repository configuration.

## Supported version

Security fixes are handled on the default branch, `main`.

## Reporting a vulnerability or sensitive issue

Please do not open a public issue for secrets, private data, supply-chain risks,
or abuse paths.

Use GitHub's private vulnerability reporting / security advisory flow if it is
available on the repository. If that is not available, contact the maintainer
through the GitHub profile linked from the repository owner and include only the
minimum detail needed to arrange a private report.

Good reports include:

- The affected file, URL, script, or workflow.
- What could go wrong and who is affected.
- Reproduction steps when safe to share.
- Whether the issue came from this repository or from an upstream example source.

## Upstream example issues

Example files in `conventions/*/examples/` are copied from public upstream
sources with provenance comments. If a copied example contains a secret,
personal data, malware indicator, or license-sensitive content, report it here
and also consider notifying the upstream owner.
